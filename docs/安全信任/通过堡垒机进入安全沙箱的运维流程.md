# 通过堡垒机进入安全沙箱的运维流程
# 一、场景

- 场景一：模型提供方在发布模型阶段可能需要通过堡垒机，进入安全沙箱进行调试
	
- 场景二：当出现安全事件或排查某些问题时，模型提供方、用户或平台方需要通过堡垒机访问安全沙箱；模型提供方和用户需要通过oncall联系值班同学获取堡垒机登陆方式
	

# 二、适用范围

- 本文档适用于所有需要申请堡垒机接入调试的用户。本文档涉及申请方和审批方双方的操作流程
	
- **任何人员登陆堡垒机后的所有行为将会被记录和审计**
	

# 三、申请流程

## 申请方（模型提供方、用户或平台方的技术支持同学）

1. 申请准备
	1. 精调：提供需要进入的“任务id+实例名称”。给oncall同学提供的信息示例：t-20240111104141-nsgxh-worker-0
		
	2. 推理
		- 提供想要进入的服务实例 ID。给oncall同学提供的信息示例：s-20240111110507-lknts-6c55544744-n4jq8
			
		- 提供ssh的账户名和密码
			

:::warning

- 需要保证推理镜像添加了sshd
	
- 为保障安全性，账号密码请点对点私发给oncall的同学，请勿直接发在oncall群
	
- 一个审批人只能审批一个堡垒机主机，若需同时运维多个pod则需要多位同学审批
:::
	

2. 审批成功后，通过oncall同学给的密码登陆堡垒机
