# 模型下线公告
随着火山方舟平台模型的升级、发布，新版本模型在效果、推理时延等各方面将远超较旧版本的模型。因此，我们将逐步下线这些旧版本模型。当某个模型确定下线计划时，我们会通过短信、站内信等方式发布通知，请及时关注。
# 下线流程
一般情况下，每个版本的模型生命周期为3~6个月，即在上线后3-6个月后，模型会正式下线并停止服务。在停止服务前，我们会为开发者预留一定的时间进行新版本模型的效果测试，避免影响业务。

1. **确定旧版本模型下线计划**，通过短信、站内信、平台公告等多种方式发布通知。
2. 自正式通知日（即「通知时间」）起，逐步下调模型的使用额度（即下调RPM和TPM的配额）。
3. 至模型停止新购（即 「 EOM 时间」 ）起，模型停止新购，新购和创建模型接入点的入口关闭，存量的模型推理接入点可正常使用。
4. 至模型下线时间（即「EOS 时间」）时，**模型正式下线并停止服务**，存量的模型推理服务将无法继续调用。

# 节点说明
## **1.模型停止新购(EOM)** 
EOM：End of Marketing，旧版模型停止新购，无法新购/新建/开通旧版模型服务。

* **不可新创建推理服务接入点**：不可再新建待下线模型的推理服务接入点，仅支持切换推理服务接入点指向的模型版本；不可新建基于待下线模型的精调任务及批量推理任务。
* 逐步下调配额：模型会逐步下调已创建推理服务接入点使用配额，直到完全下线。

:::tip
* 对于自定义推理接入点
   * 若您选择在控制台手动切换，请确保在切换前已在开通管理模块开通替代模型，否则切换将报错，但不影响线上服务。
   * 若您选择平台自动切换，请确保下线日前在开通管理模块开通替代模型，否则会出现调用报错，**可能影响线上服务**。
* 对于预置推理接入点
   * 到期将直接关停，不自动切换，也不支持在控制台手动切换。请您在模型下线前，及时做好相关业务调整，以免影响正常使用。
:::
## **2.模型下线 (EOS)** 
EOS：End of Service & Support，模型停止服务，模型服务下线。

* 精调模型：模型下线当日停止服务，无法调用。
* 非精调模型：模型下线当日系统将自动切换到新模型服务或停止服务，详见下文对应批次的模型下线说明；若自动切换的模型在正式切换时刻有最新版本，会变更至最新版本。

:::tip
若下线日前未手动迁移，系统自动迁移的模型对比原模型可能会有价格波动，请关注价格变化，对比价格差异，以便合理规划使用成本。
:::
# 切换模型步骤
模型切换涉及到不同场景的切换，包括同模型不同版本、不同模型迁移，精调模型服务迁移；以下是针对不同场景的迁移指引。
## 场景：基础模型切换
平台提供**推理服务接入点平滑切换能力。** 在方舟平台的在线推理页面，修改推理服务接入点对应的模型版本（暂不支持精调模型及模型单元的版本切换）。可实现模型版本的平滑切换，支持更新模型版本。
:::warning
不同模型/版本支持的能力、API 参数、单价、任务中实际表现存在差异，迁移前请务必仔细阅读模型介绍，并进行必要评测，比较迁移前后模型的表现。
:::

## 场景：精调模型切换
由于精调模型的依赖基座模型，如果需要变更精调模型对应的基座模型，需要在新的模型版本上再次[精调模型](/docs/82379/1099459)，并验证效果达到预期后，再创建新的推理接入点，完成相应的代码变更，实现迁移。

如有其他迁移问题，可以通过提交[工单](https://console.volcengine.com/workorder/create?step=2&SubProductID=P00001166)的方式，进行咨询。
