# 知识库配额说明
## 旗舰版
## 知识库配额限制

| | | | \
|**类别** |**最高** |**描述** |
|---|---|---|
| | | | \
|知识库 |200 个 |单个账户最多创建知识库数量 |
| | | | \
|文档数量 |300万 个 |单个知识库最多上传文档数量 |
| | | | \
|切片数量 |10万 个 |单个文档支持最大切片数 |
| | | | \
|CPU 配额 |100 CU |单个知识库支持的最大配额 |
| | | | \
|QPS 上限 |1万 QPS |单个知识库支持的最大检索 QPS，与知识库所配 CPU 核成正比，一般 1 CPU 核约支持 100 QPS 检索 |
| | | | \
|向量维度 |4096 维 |与所选向量化模型有关，最高支持 4096 维 |
| | | | \
|文本向量化模型 |自定义 |支持 Doubao-embedding+多功能版、Doubao-embedding、短文本高精度+多功能版、多功能版、短文本高精度版 |
| | | | \
|切片最大长度 |8000 tokens |与所选向量化模型有关，最大支持 8000 tokens |
| | | | \
|标签 |180 个 |单个知识库支持最大标签数 |
| | | | \
|检索标签值数量 |1024  |单次检索时，单一标签可指定的检索值数量 1024 |
| | | | \
|单次上传文档数量 |100 个 |单个知识库**前端一次本地上传**的最大文档数量，如需更大规模上传建议使用 TOS |
| | | | \
|文档上传及处理速度 |约 8万 个/天（小文件） |单个知识库可处理的文档速度，与所传文档大小和文本向量化模型相关 |
| | | | \
|召回数量 |200 个 |支持检索召回的最大文本片数量 |
| | | | \
|结构化文档表字段数量 |380 个 |单个知识库支持的最大表字段数量 |
| | | | \
|Rerank 接口模型限流 |15 万tokens/秒 |Rerank 模型接口所能支持的最大流量上限 |\
| | |> 如需提高流量上限，可联系火山客服咨询 |
