# 订单管理

## 概述

### 1. 下单介绍

登录各产品控制台可进行相应产品选配以及下单，具体下单流程和说明可参考各产品的相关文档。

### 2. 下单余额限制

当前平台对于下单会根据不同商品进行不同的余额限制，具体规则如下：

- 若购买的商品为**预付费商品**，则需要保证账户未欠费才可下单，或者在支付时将欠款补齐后方可下单；
- 若购买的商品为**后付费商品**，且**结算周期为月结**，则需要先补齐欠款后才可下单；
- 若购买的商品为**后付费商品**，且**结算周期为小时结或日结**，则下单余额门槛为0元或者100元。
  - 如商品为计量型配置，则下单余额门槛为0元，即无欠费或有欠费但补齐欠款后，方可下单（具体可参考对应商品的计费说明文档）；
  - 如商品为周期型配置，则下单余额门槛为100元，即无欠费或有欠费但补齐欠款后，并且可用余额+代金券的总和满足余额门槛，方可下单；（为了避免下单开通后，短期内余额就被消耗完毕导致欠费关停，故设置了100元的门槛）。

### 3. 价值分摊规则

在更改预留实例券的场景下，需要将原券的剩余价值，按照目标券的计算力占比进行分摊，给每张目标券分摊相应的价值。

- 分摊后的预留实例券价值，将在后续更改、退订时使用；
- 具体分摊规则如下：
  - 第一步：计算原券的剩余价值：
    - 将所有原券的剩余价值分别计算，最终求和得到原券的所有剩余价值；
    - 原券剩余价值计算可参考[退订管理](../退订管理/退订管理说明.md)中云服务器的金额计算公式。
  - 第二步：分摊价值至目标券：
    - 每一张目标券的价值=目标券的计算力/所有原券的计算力总和*所有原券的剩余价值总和；
    - 其中：每张预留实例券的计算力=计算力因子*实例数量，计算力因子由该券的配置规格决定。

## 费用中心-订单管理介绍

登录火山引擎控制台，进入**费用中心-订单管理**，可以查看所有创建的订单信息，还可以对已创建待支付的订单进行支付或取消。

### 1. 查询和筛选条件

可以根据以下查询和筛选条件查询订单。

- **订单号**：输入订单号查看指定订单。
- **创建时间**：选择起始日期和截止日期查看属于此时间段创建的订单。默认查询近1个月的订单。
- **产品**：选择产品筛选对应产品的订单。
- **订单类型**：选择订单类型筛选不同订单类型的订单。
- **订单状态**：选择订单状态筛选不同订单状态下的订单。

### 2. 表格字段说明

订单列表展示订单号、产品、类型、创建时间、更新时间、状态、原价、应付等信息。

- **订单号**：订单的唯一标识。
- **产品**：显示订单对应的产品名称。
- **类型**：表示订单的类型，有"新购、试用、转正、续费、更配、退订、预留实例调整、临时升配"这8种。
- **创建时间**：代表订单创建的时间。
- **更新时间**：代表状态更新的时间。
- **状态**：代表订单的当前状态，有"待支付、已支付、已取消、已退款、部分退款、退款中、退款失败"这7种。
- **原价**：代表订单的原价金额。
- **应付**：代表订单中需现金支付的金额。
- **操作**：针对状态为"待支付"的订单，可执行"支付、取消"的操作；针对其他状态的订单，仅可点击订单号"查看详情"的操作。

### 3. 订单详情

点击订单号，可打开订单详情页面，分成两种展示情况。

**3.1. 新购、续费、试用、转正类型的订单详情页面**

**3.2. 更配、退订、预留实例调整、临时升配类型的订单详情页面**

#### 订单概览字段说明

- **订单号**：订单的唯一标识。
- **类型**：显示订单对应的订单类型。
- **创建时间**：显示订单创建的时间。
- **付款方式**：显示订单付款方式是预付费或后付费。
- **状态**：代表订单的当前状态。
- **原价合计**：显示订单详情中各子订单的原价金额之和。
- **优惠金额**：显示订单详情中各子订单的优惠金额之和。
- **代金券抵扣**：显示订单详情中各子订单的代金券抵扣金额之和。
- **应付金额**：显示订单详情中各子订单的应付金额之和。
- **实付金额**：显示订单详情中各子订单的实付金额之和。

#### 订单详情字段说明

订单详情用于展示每笔订单行为相关的交易信息。每笔订单都会有相应的订单明细，对于更配、预留实例调整、连续临时升配订单，需要展示变更前的老配置信息与变更后的新配置信息，故会出现多条订单明细记录。

完成订单下单后，基于本次订单交易行为，需要单独记录本次资源服务交付过程，该资源服务被称之为"实例"。

- **产品**：显示订单详情中各子订单对应的产品名称。
- **配置详情**：显示订单详情中各子订单对应的产品的具体配置参数。
- **付款方式**：显示订单详情中各子订单对应的付款方式。
- **购买时长**：显示订单详情中各子订单对应的购买时长，预付费会有此次下单购买的时长，后付费则无此信息。
- **起止时间**：显示订单详情中各子订单对应的购买时长的起止时间，预付费会有此次下单购买的起止时间，后付费则无此信息。
- **实例ID**：显示订单详情中各子订单对应的产品配置交付后创建的实例唯一标识。
- **状态**：显示订单详情中各子订单对应的的当前状态，有"待支付、已支付、已取消、已退款、退款中、退款失败"这6种。
- **原价金额**：代表子订单的原价金额。
- **优惠金额**：代表子订单的合同价或促销活动产生的优惠金额。
- **代金券抵扣**：代表支付时用代金券抵扣分摊到子订单上的代金券抵扣金额。
- **应付金额**：代表子订单需现金支付的金额，应付金额=原价-优惠金额-代金券抵扣。
- **实付金额**：代表子订单实际现金支付的金额，支付完成后，实付金额=应付金额。
